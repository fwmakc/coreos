# Track 2: Core Shell

## Цель

Интерфейс поверх Core Runtime. Вместо эксплорера и свалки документов — чистая работа с идеями и проектами. Облако тегов и связанности. Управление через поисковую строку.

---

## Концепция

### Классический десктоп
```
Рабочий стол → Папки → Файлы → Приложения (каждое в своем окне)
```

### Core Shell
```
Поиск (Cmd+K) → Проекты → Идеи/Задачи/Файлы (связанные тегами)
```

Нет "рабочего стола" как такового. Есть пространство, в котором живут проекты. Каждый проект — контейнер, внутри которого всё связано: заметки, файлы, задачи, ссылки, чаты.

---

## Ключевые элементы Shell

### 1. Command Bar (Cmd+K)

Центральный элемент интерфейса. Вызывается из любого места.

**Возможности:**
- Поиск по проектам, заметкам, файлам, тегам
- Запуск приложений: "открой калькулятор", "новая заметка"
- Создание сущностей: "создай проект X", "добавь тег Y"
- Навигация: "перейди к проекту Z"
- Голосовой ввод: говоришь — строка заполняется

**UX:**
- Появляется по центру экрана (как Spotlight/Raycast)
- Топ-3 результата + preview
- Стрелки для навигации, Enter для выбора
- ESC для закрытия

### 2. Projects (Проекты)

Проект — контейнер, внутри которого всё связано контекстом.

**Структура проекта:**
```
Project: "CORE OS"
├── Ideas (заметки-идеи, связанные тегами)
├── Tasks (задачи с статусами)
├── Files (прикрепленные файлы)
├── Links (ссылки на ресурсы)
└── Tags: [os, rust, webgpu, mvp]
```

**Взаимодействие:**
- Клик на проект → раскрывается как пространство
- Всё внутри проекта связано: идея → задача → файл
- Переключение между проектами: свайп / Cmd+1-9 / через Command Bar

### 3. Tags (Облако тегов)

Теги — главный инструмент навигации. Не папки, а связи.

**Как работает:**
- Любая сущность (идея, файл, задача) имеет теги
- Фильтр по тегу: показываются все сущности с этим тегом из ВСЕХ проектов
- Связи: тег "rust" → видны идеи из "CORE OS" + задачи из "Учеба" + файлы из "Работа"
- Визуализация: облако тегов (размер = частота использования)

**Теги vs Папки:**
- Файл в одной папке = в других не видно
- Файл с тегами = виден из любого контекста
- Один файл = много тегов = много точек входа

### 4. Ideas (Идеи/Заметки)

Минимальный текстовый блок. Быстрое создание, быстрый поиск.

**Формат:**
- Заголовок (обязательный)
- Текст (markdown)
- Теги
- Связи с другими идеями/задачами

**Создание:**
- Cmd+K → "новая идея" → сразу редактор
- Голосом: "запиши идею: использовать WebGPU для рендеринга"
- Из буфера обмена: "сохрани как идею"

### 5. Timeline (Лента активности)

Хронологический поток всех действий:
- Создана идея
- Добавлен файл
- Завершена задача
- Написана заметка

Позволяет "отмотать" назад и вспомнить, что делал.

## Интеграция ИИ (MVP)

### Голосовой ввод
- Локальный Whisper (через WebGPU или ONNX)
- Команды: "найди...", "создай...", "открой..."
- Транскрипция в Command Bar

### Умный поиск
- Полнотекстовый поиск по всем заметкам, файлам, тегам
- Fuzzy matching (опечатки)
- Контекст: если открыт проект "CORE OS" — результаты из него в приоритете

### В MVP НЕ входит
- Generative UI
- ИИ-ассистент как чат-бот
- Автоматическое тегирование
- Semantic search (векторный поиск)

## Визуальный стиль (MVP)

### Принципы
- Минимализм: только необходимое
- Темная тема по умолчанию
- Скругленные углы, мягкие тени
- Шрифт: Inter (или системный sans-serif)
- Акцентный цвет: один (голубой/зеленый)

## Milestones

### Sprint 1 (Месяц 1): "Макеты и заглушки"

**Цель:** Дизайн-система + прототип Shell на любом доступном средстве (HTML/Figma/Bun).

- [ ] Wireframes всех экранов (TopBar, Sidebar, Content, Command Bar)
- [ ] Цветовая палитра + типографика
- [ ] Компоненты: View, Text, Button, TextInput, ScrollView (в макетах)
- [ ] Прототип навигации: переключение проектов, открытие идей
- [ ] Прототип Command Bar: ввод → фильтрация → результат
- [ ] Спецификация данных: структура проекта, идеи, тега (JSON schema)

**Критерий успеха:** Можно "покликать" прототип и понять, как будет работать.

### Sprint 2 (Месяц 2): "Живой Shell"

**Цель:** Shell работает на Core Runtime (Трек 1).

- [ ] TopBar: логотип, Command Bar триггер, настройки
- [ ] Sidebar: список проектов + облако тегов
- [ ] Content Area: отображение идей как карточек
- [ ] Переключение проектов (клик в Sidebar)
- [ ] Command Bar: открытие, ввод, фильтрация (пока по названиям)
- [ ] Базовые анимации: открытие/закрытие, переходы

**Критерий успеха:** Shell рисуется через WebGPU, навигация работает.

### Sprint 3 (Месяц 3): "Проекты и голос"

**Цель:** Полный цикл: создать проект → добавить идею → тегировать → найти через поиск/голос.

- [ ] CRUD для проектов, идей, тегов (хранение в SQLite через Bun)
- [ ] Полнотекстовый поиск по идеям
- [ ] Фильтрация по тегам
- [ ] Timeline (лента активности)
- [ ] Голосовой ввод: Whisper → текст → Command Bar
- [ ] Демо-сценарий: полный цикл работы

**Критерий успеха:** Голосом создать проект, добавить идею, найти через поиск.

## Data Model (MVP)

```typescript
interface Project {
  id: string;
  name: string;
  description: string;
  tags: string[];
  createdAt: number;
  updatedAt: number;
}

interface Idea {
  id: string;
  projectId: string;
  title: string;
  content: string;       // markdown
  tags: string[];
  linkedIds: string[];   // связи с другими идеями/задачами
  createdAt: number;
  updatedAt: number;
}

interface Tag {
  id: string;
  name: string;
  color: string;
  usageCount: number;
}

interface Activity {
  id: string;
  type: 'idea_created' | 'idea_updated' | 'project_opened' | 'tag_added';
  entityId: string;
  entityType: 'project' | 'idea' | 'tag';
  timestamp: number;
}
```

### Storage
- SQLite через Bun (встроенный)
- Одна БД на пользователя
- Расположение: `~/.core/data.db` (или `%APPDATA%/core/data.db`)
