# План реализации CORE OS

> Этот документ описывает этапы разработки CORE OS от фундамента до готовой системы. Каждый этап — самодостаточная единица: после его завершения систему можно собрать и протестировать.

---

## Принцип разделения

Этапы упорядочены **снизу вверх** по архитектуре:

```
Этап 10: Performance & Scale (Оптимизация, стресс-тесты, CI/CD)
Этап 9:  Релизная полировка (Game Mode, Energy, Store, Migration, Docs)
Этап 8:  Безопасность и администрирование (RBAC, Audit, Key Manager, Admin UI)
Этап 7:  Voice Engine (Whisper, TTS, Zero UI, Intent Queue)
Этап 6:  Communication Layer (Messenger, Email, VoIP, AI-мост)
Этап 5:  P2P Mesh и синхронизация (WireGuard, CRDT, Sync Engine)
Этап 4:  App Runtime (Island Mode, V8 Isolates, App Registry, Permissions)
Этап 3:  Проекты и окна (Project Manager, Window Manager, Layout)
Этап 2:  Command Bar (Input Router, Suggestion Engine, 8 режимов)
Этап 1:  Фундамент (Host Shim + Display Server)
```

**Ключевое правило:** этап N не требует функциональности из этапов > N. Placeholder-реализации допустимы, если они явно помечены и заменяются на полноценные в следующих этапах.

## Технологический стек (константа для всех этапов)

| Компонент | Технология | Назначение |
|-----------|-----------|-----------|
| Host Shim | Rust | Окно, ввод, GPU, файлы, сеть, аудио |
| Display Server | Rust + wgpu | WebGPU рендеринг, композитинг |
| Micro-Kernel | Bun (TypeScript) | Командная строка, проекты, runtime |
| App Runtime | V8 Isolates (via Bun) | Приложения, sandbox |
| AI Engine | llama.cpp / Ollama + Bun | Whisper, SLM, TTS |
| Sync | WireGuard + libp2p + CRDT | P2P сеть, синхронизация |
| Storage | SQLite (Bun built-in) | Данные, индексы, настройки |

## Формат файла этапа

Каждый файл `phase-NN-*.md` содержит:

1. **Цель** — что должно работать после этапа
2. **Зависимости** — от чего зависит (только предыдущие этапы)
3. **Компоненты** — что реализуем
4. **Технические требования** — детали API, схемы, форматы
5. **Шаги реализации** — порядок работы внутри этапа
6. **Критерии приёмки** — как проверить, что этап готов
7. **Placeholder'ы** — что временно заменено заглушкой
8. **Cross-reference** — связь со слоями
