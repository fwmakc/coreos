# CORE OS — Бэк-офис и Суперюзер

## Принцип: "Цифровой холодильник"

Бэк-офис — не сервер. Это бытовой прибор: включил в розетку один раз и забыл. Он сам себя обновляет, чинит и бэкапит.

---

## Три уровня Бэк-офиса

### 1. Персональный (Home/Solo)
Домашний сервер или постоянно включенный ПК.

**Функция:** Хранилище тяжелого контента + личный узел синхронизации.

**Фишка:** Если домашний комп выключен, а VDS есть — VDS отдаст свежий кеш документа.

### 2. Корпоративный/Школьный (Local Cloud)
В серверной офиса или школы.

**Функция:** Локальное облако на скоростях гигабитной локалки.

**Фишка:** Общие папки для сотрудников. Админ настраивает один раз — у всех в CORE автоматически появляется диск "Work".

### 3. Федеративный (Global Namespace)
Разные источники данных → единое дерево:

```
storage/
├── local/          (память устройства)
├── home-server/    (бэк-офис в квартире)
├── office/         (сервер на работе)
└── cloud-paid/     (купленное облако)
```

Для приложений — один диск. Система сама разруливает маршруты и шифрование.

## Уровни доступности

### Уровень "Гик/Энтузиаст" (Soft-only)
- Образ системы или Bun-пакет
- Сам выбирает железо: старый ноут, Raspberry Pi, VDS
- Core Certified: если сборка проверена сообществом

### Уровень "Масс-маркет" (Branded Hardware)
- Эталонная железка: крошечный, бесшуный неттоп
- Купил → воткнул в розетку и роутер → готово
- Core-Plug: коробочка размером с зарядку, $49
- Основная маржа здесь

### Уровень "Экосистема" (China/OEM)
- Открытые спецификации (Hardware Spec)
- Китайские заводы клепают Core-совместимые NAS, роутеры, планшеты
- Масштабирование без своих заводов
- Роялти за ОС

### "Два клика" для большинства
Первый Бэк-офис стоит на той же машине, где CORE:
1. Клик 1: установил Shell (графику)
2. Клик 2: активировал Бэк-офис (фоновую службу)

Когда юзер созреет до железки → кнопка "Перенести данные на внешнюю ноду".

## Суперюзер (The Owner)

### Концепция: убить "админку"
Обычный пользователь не знает слов "IP-адрес", "порт", "права доступа".

### Суперюзер = цифровой ключ
Абсолютная власть над всей сетью. Управление вшито прямо в Shell CORE.

### Что делает Суперюзер
- **Добавление пользователей:** "Поделиться доступом" → QR-код
- **Ограничение объема:** Ползунок на иконке диска
- **Права доступа:** Перетащить папку на контакт = "Вася может смотреть эти фото"
- **Слияние ресурсов:** "Считай старый ноут и новый Бэк-офис одним диском"

### Без паролей (Biometric Handshake)
Вход подтверждается отпечатком/FaceID на телефоне. Бэк-офис "узнает" по P2P-каналу.

---

## ИИ-Админ (невидимый инженер)

Бэк-офис сам следит за собой:
- Обновляет софт в безопасное время
- Настраивает зеркалирование (RAID) при обнаружении двух дисков
- Блокирует подозрительные входы: "Кто-то из Китая ломится — я его забанил. Ты же не в Шанхае?"
- Место заканчивается → человеческое уведомление: "Давай почистим кэш или подключим еще диск"

### Self-Healing
При критической ошибке:
- Откат к Golden Image (Read-only SquashFS)
- Защищенный сектор памяти
- Перзагрузка без потери данных

## Подключение нового Бэк-офиса

### Процесс
1. Сканируешь QR-код в админке сервера через CORE
2. Бэк-офис появляется как новый диск
3. Система сама настраивает шифрование и маршрутизацию

### Восстановление доступа
При потере главного девайса:
- Recovery-ключ (записан при первой настройке)
- Подтверждение через другое доверенное устройство

---

## Монетизация Бэк-офиса

1. **SaaS:** Продажа места в официальном облаке (для тех, кто не хочет возиться с серверами)
2. **Self-hosted Pro:** Бесплатно для дома, платно для компаний (админка, аудит, бэкапы)
3. **Hardware:** Наценка на Branded Hardware
4. **Маркетплейс:** Комиссия с платных модулей/приложений

## Промышленный кейс (Ноябрьск)

### Для Газпрома и аналогов
- Бэк-офис в серверной на месторождении
- P2P-сеть между устройствами рабочих (без спутника)
