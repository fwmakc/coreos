# CORE OS — Виртуальная файловая система (VFS)

## Принцип: привычные папки с суперсилой

Нельзя прийти к человеку и сказать "забудь про папки". Но можно взять старую форму и наполнить её новым смыслом.

---

## Уровень 1: Виртуальная абстракция (VFS)

Для пользователя это папки. Физически — объекты в базе данных.

### Теневое копирование
Файл в "папке" — это индексированная сущность в базе метаданных.

### Сквозные связи
Один файл лежит в "Проекты" и в "Важное" одновременно, не дублируя место на диске. Это не "ярлыки", а разные отображения одного объекта (теги, которые выглядят как папки).

---

## Уровень 2: Smart Folders (Папки с интеллектом)

Папка, которая сама себя наполняет по правилу:
- "Сюда попадают все PDF-отчеты от бухгалтерии за март"
- Система сама сканирует входящие файлы, почту, мессенджеры
- Учитель: папка "10-А класс" наполняется работами учеников автоматически

---

## Уровень 3: Версионность (Time Machine для каждого файла)

Файловая система работает как Git:
- Каждое сохранение — микровселенная
- Клик правой кнопкой на файл — ползунок отката на 5 минут / вчерашний вечер
- Конец файлов `отчет_финал_v2_ред_Ивана.docx` — просто `отчет.docx` со всей историей

---

## Уровень 4: Контекстный поиск (Deep Indexing)

### Поиск по смыслу
- Запрос "смета на ремонт" — система находит скриншот с распознанным текстом
- Контекст: открыта папка "Математика" — приоритет формулам и графикам

### Индексация
- В фоне на Bun/JS
- Распознавание текста на изображениях
- Индексация содержимого документов

---

## Разделение "Тела" и "Паспорта"

### Паспорт (DB — SQLite через Bun)
Хранятся только метаданные: название, размер, теги, превью, хэш-сумма. Весит копейки. Мгновенный поиск по 2 ТБ.

### Тело (Blob Storage)
Сами гигабайты лежат на жестком диске. Система знает: "хэш X = файл по пути D:/Movies/Inception.mkv".

---

## Content Addressable Storage

- Файловая система на Merkle DAG (как в IPFS)
- Файл = хэш (CID — Content Identifier)
- Дедупликация: 3 копии фильма по 50 ГБ = 1 копия на диске + 3 точки входа
- Один файл у двоих в локалке — тянется по P2P, а не через интернет

---

## Зеркальные папки (Интеграция с хост-ОС)

Поскольку CORE ставится поверх Windows/Mac, делаем "Зеркальные папки":
- Внутренняя папка CORE привязана к реальной папке на диске D:/Work
- Всё, что делается в CORE, синхронизируется с реальной ФС винды и наоборот
- Офисный работник работает в красивой среде, но файл физически там, где его ожидает увидеть Windows

---

## Федеративная файловая система

Разные источники данных объединяются в единое дерево:

```
storage/
├── local/          (память телефона/ноута)
├── home-server/    (бэк-офис в квартире)
├── office/         (сервер на работе)
└── cloud-paid/     (купленное облако для бэкапов)
```

Для приложений это один диск. Система сама разруливает маршруты и шифрование.

---

## Ленивая загрузка (On-Demand)

- На телефоне видна вся медиатека, но файлы весят 0 байт ("призраки")
- Нажал Play — домашний Core-сервер стримит поток прямо в рантайм
- Умный кэш: система видит, что ты слушаешь один плейлист — сама скачивает для офлайна

---

## P2P-стриминг

Твой домашний комп = твое личное облако:
- Устройства объединяются в частную P2P-сеть (WireGuard + libp2p)
- Зашифрованный туннель из любой точки мира
- Хард на 2 ТБ как бы "вставлен" прямо в телефон

### Если домашний комп выключен
- Плавно: "Этот файл сейчас недоступен (оффлайн), показать последнюю копию?"
- Background Fetch: раз в 15-30 мин подтягиваются обновления мессенджеров
- Облачный буфер для критичных данных (опционально)
