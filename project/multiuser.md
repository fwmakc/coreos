# CORE OS — Мультипользовательность

## Принцип: "Один Движок — Много Личностей"

В классических ОС переключение пользователя — тяжелый процесс с завершением сессий. В CORE — как переключение вкладок в браузере.

---

## Архитектура

```
Core-процесс (Rust/Bun рантайм) — запускается один раз
  ├── Sphere: "Work" (V8 Isolates, свои данные)
  ├── Sphere: "Personal" (V8 Isolates, свои данные)
  └── Sphere: "Anonymous" (V8 Isolates, временные данные)
```

### Сфера (Sphere)
- Свой набор V8 Isolates для приложений
- Свой зашифрованный контейнер данных
- Физически разделенная память

---

## Мгновенное переключение (Hot Swap)

- Переключение между профилями = переключение вкладок в браузере
- Можно держать открытыми "Рабочий" и "Личный" профиль одновременно
- Каждый профиль зашифрован своим ключом
- При переключении — расшифровка "на лету" только данных текущего профиля

### В интерфейсе
Панель (Sidebar) с иконками профилей. Клик на "Home" → весь интерфейс, приложения и тема меняются за 100мс.

---

## Роли (Contextual Identities)

Вместо классических "Администратор" и "Гость":

| Роль | Описание |
|------|----------|
| **Work** | Рабочие инструменты, корпоративные API, история только по работе |
| **Dev** | Полный доступ к терминалу и системным вызовам |
| **Personal** | Личные данные, развлечения, соцсети |
| **Anonymous/Sandboxed** | Временный юзер. Закрыл окно — все данные стерлись навсегда. Идеально для тестирования сомнительных пакетов |

## Freeze & Sync (Заморозка состояния)

### Проблема
5 открытых профилей с 20 мессенджерами жрут ресурсы.

### Решение: Snapshots
1. **Заморозка:** При переключении профиля система делает Heap Snapshot всех приложений → в оперативку или на быстрый SSD
2. **Процессор:** Не выделяется для замороженных профилей
3. **Событийный шлюз:** Один сверхлегкий системный сервис слушает WebSocket/Push в фоне
4. **Пробуждение:** Разворачивание дампа за миллисекунды → приложения опрашивают шлюз: "Что я пропустил?"

### Background Fetch
Раз в 15-30 минут система "приоткрывает" замороженный профиль:
- Мессенджеры подтягивают данные в локальную базу
- Снова отправляются в сон
- Пользователь заходит — всё уже актуально

### Исключительный режим
Тяжелые задачи (рендер видео, скачивание):
- Приложение: `OS.requestBackgroundMode()`
- Система: иконка "Юзер А что-то делает в фоне"
- Выделяются кванты процессорного времени

## Общие ресурсы (Shared Immutable Core)

### Принцип
Ядра приложений (бинарный код, базовые библиотеки) лежат в Shared Storage (только чтение).

- Состояние (State), конфиги и данные — у каждого свои
- Экономия места на диске
- Моментальный запуск

---

## Безопасность данных

### Каждый профиль = зашифрованный контейнер
- SQLite или специализированный файл
- Мастер-пароль / биометрия при входе
- Кража физического диска → злоумышленник видит зашифрованный мусор

### Shared Storage (только чтение)
- Общие библиотеки доступны всем профилям
- Модификация только Суперюзером
- Изоляция: один профиль не может читать данные другого

---

## Суперюзер (The Owner)

- Управляет всеми профилями из своего Shell
- Добавляет пользователей: "Поделиться доступом" → QR-код
- Ограничивает объем: ползунок на иконке диска
- Настраивает права: перетаскивание папки на контакт = "Вася может смотреть эти фото"

### Восстановление доступа
При потере главного девайса:
- Recovery-ключ (записан при первой настройке)
- Подтверждение через другое доверенное устройство
