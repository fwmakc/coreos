# CORE OS — Режимы интеграции с хост-ОС

## Три режима работы

CORE не заставляет выбирать один режим. Режимы переключаются динамически в зависимости от контекста.

---

## Level 0: Overlay (Фоновый режим)

CORE "прозрачна". Видны окна хост-ОС, но наши виджеты плавают сверху.

### Поведение
- Ввод (мышь/клава) проходит сквозь CORE в хост-ОС
- Клик на наш виджет — перехватывается CORE
- ИИ-ассистент доступен через жест/горячую клавишу

### Применение
- Быстрые заметки поверх Windows
- Мини-плеер в углу экрана
- Виджет погоды/уведомлений

---

## Level 1: Managed Space (Рабочий режим)

CORE захватывает экран и работает как оконный менеджер.

### Поведение
- Приложения просят у Ядра: "Дай мне прямоугольник 800x600"
- CORE сама рисует окна, управляет размером
- Хост-ОС не видна (заблюрена под полупрозрачным слоем)

### Применение
- Работа с документами
- Мессенджеры
- Файловый менеджер
- Повседневные задачи

---

## Level 2: Full Access / Exclusive (Игровой/Профессиональный режим)

Приложение запрашивает `requestExclusive()` → CORE отдает ему контекст видеокарты и устройства ввода.

### Поведение
- Display Server перестает рисовать панель задач и другие окна
- 100% ресурсов и пикселей отдано приложению
- Прямой туннель ввода: все нажатия летят сразу в приложение (минимальный Input Lag)
- Panic Gesture: три пальца вверх / Hold Escape — принудительный возврат в Shell

### Применение
- Тяжелые игры
- Видеоредакторы
- 3D-моделирование
- Стриминг

---

## Бесшовные переходы (Spatial Consistency)

Ключевая фишка: переходы между режимами — не "закрыть/открыть", а морфинг.

### Пример сценария
1. Плеер висит как виджет в углу (Overlay)
2. Тянешь за край → морфирует: дорисовываются кнопки, список треков, эквалайзер (Managed)
3. Нажал "Полноэкранный режим" → весь экран отдан плееру (Exclusive)
4. Вышел → плавно вернулся в Managed → свайпом свернул в Overlay

### Техническая основа
- **Shared WebGPU Context:** При переходе в Exclusive графический контекст не реинициализируется
- **Canvas Resize:** Смена режима = смена размера холста (мгновенно, без черных экранов)
- **Morphing UI:** UI-компонент переключает режим отображения (minimized → full → exclusive)

## Z-слои (Умная глубина)

```
┌─────────────────────────────────┐
│  Слой ИИ-ассистента (самый верх)│  "Смотрит" на всё
├─────────────────────────────────┤
│  Слой Приложений CORE           │  Летают над Core Space
├─────────────────────────────────┤
│  Слой Core Space (полупрозрачн.)│  Матовый слой наплывает на хост-ОС
├─────────────────────────────────┤
│  Слой Windows/Хост-ОС (самый низ)│
└─────────────────────────────────┘
```

- Переход в Exclusive → остальные слои в `opacity: 0` и `blur`, но не выгружаются из памяти
- Зажал системную клавишу → "подсмотрел" что внизу

## Delegation Model (Управление экраном)

CORE выступает как супер-визор (диспетчер):

### Display Server как прокси
Приложение не общается с видеокартой напрямую. Оно шлет команды Display Server:
- Managed: Display Server рисует окна + панель задач
- Exclusive: Display Server отключает всё лишнее, отдает 100% пикселей приложению

### Перехват ввода (Input Hooking)
- **Managed:** Ядро ловит нажатия → "Cmd+Tab — мне, буква A — текстовому редактору"
- **Exclusive:** Прямой туннель в приложение (минимизация задержки)
- **Panic Gesture:** Всегда доступен для принудительного возврата в Shell

---

## "Слой-Паразит" (Overlay Hybrid)

CORE создает невидимый слой поверх всей Windows:
- Shell (панель задач, виджеты, ИИ) "плавают" поверх винды
- Нажал "Войти в CORE" → экран плавно заблюривается → развертывается наше пространство
- Можно "вытащить" окно из CORE → оно станет обычным окном Windows
- Можно "затянуть" окно хост-ОС в CORE

### Техническая реализация
- Прозрачность (Alpha Channel) на системном уровне
- Layered Windows API в Windows
- Проще в Linux (Wayland) и macOS
