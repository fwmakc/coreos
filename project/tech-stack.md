# CORE OS — Стек технологий

## Обоснование выбора

CORE OS использует гибридный "сэндвич": каждый язык на своем месте.

```
┌──────────────────────────────────┐
│  Приложения (сторонние)          │  TypeScript / JS (V8 Isolates)
├──────────────────────────────────┤
│  Системные API                   │  TSCLANG (TS → Native через LLVM)
├──────────────────────────────────┤
│  Рантайм приложений              │  Bun (V8 + встроенный SQLite/TLS)
├──────────────────────────────────┤
│  Низкий уровень                  │  Rust (память, потоки, сеть)
└──────────────────────────────────┘
```

---

## Rust — фундамент (Level 0)

### Где используется
- Host Shim (прослойка к хост-ОС)
- Сетевой стек (eBPF, XDP)
- Thread management (Core Pinning, SCHED_FIFO)
- Memory Bridge (Zero-copy ABI)
- P2P-туннели (WireGuard bindings)

### Почему Rust
- Memory safety без GC
- Предсказуемая производительность
- Лучший FFI для интеграции с V8
- Экосистема wgpu-native, cpal, libp2p

---

## Bun — рантайм (Level 1)

### Где используется
- Исполнение JS/TS приложений
- Встроенный SQLite для метаданных
- HTTP/TLS для сетевых операций
- Менеджер пакетов

### Почему Bun, а не Node.js
- В 3-5 раз быстрее на I/O операциях
- Встроенный SQLite (не нужен внешний)
- Нативная поддержка TypeScript
- Быстрый запуск (критично для V8 Isolates)

---

## TSCLANG — системный мост (Level 1, критические модули)

### Где используется
- Системные API (VFS, Display Server internals)
- Модули обработки данных в реальном времени
- Промышленные коннекторы (Modbus, OPC UA)
- Графический движок (критические пути рендеринга)

### Что это
TypeScript-синтаксис, компилируемый в нативный машинный код через LLVM/Clang.

### Ключевые возможности
- Ручное управление памятью / ARC — нет GC
- Прямой маппинг C-типов (structs, pointers)
- SIMD и векторизация
- Бесшовная интеграция с C/C++ библиотеками

### Почему TSCLANG, а не чистый Rust
- Единый синтаксис TS для всей системы
- Разработчикам на JS не нужно учить новый язык
- Модуль можно ускорить простым флагом компиляции

### Риск
- Проект экспериментальный
- Решение: один R&D-специалист на стыке TSCLANG ↔ Rust

---

## V8 Isolates — песочница приложений (Level 2)

### Где используется
- Запуск каждого стороннего приложения
- Изоляция памяти
- Resource quotas

### Конфигурация
```
v8::Isolate::SetResourceConstraints:
  - max_old_generation_size: определяется квотой приложения
  - max_young_generation_size: определяется квотой приложения
  
v8::Isolate::TerminateExecution() — при превышении лимитов
```

### ABI-мост между TSCLANG и V8

**Dual-View Objects:**
- Объект создается в Off-heap памяти (вне GC V8)
- Структура: жесткий C-layout
- V8 получает `v8::External` или `ArrayBuffer` на ту же область памяти

**Handle Ownership:**
- В JS: обычный объект (геттеры/сеттеры)
- Под капотом: Reference Count (ARC) на нативный объект
- GC V8 собирает обертку → деструктор уменьшает счетчик
- При потере ссылки — штатное исключение Runtime-чекера (не сегфолт)

**Cost:** 2-3 наносекунды на проверку Handle vs миллисекунды на маршалинг

---

## WebGPU / Skia — графика (Level 3)

### Где используется
- Display Server
- Отрисовка всего UI
- Аппаратное ускорение рендеринга

### Выбор
- **WebGPU (wgpu-native):** Основной путь. Прямой доступ к GPU
- **Skia:** Фоллбэк / дополнительные графические операции

### Почему не Chromium/DOM
- DOM = дерево из тысяч объектов, изменение пикселя → пересчет страницы
- WebGPU = прямые инструкции видеокарте: "нарисуй прямоугольник здесь"
- 10-20x быстрее DOM
- Потребление памяти в 5-10x меньше Electron

---

## WireGuard + libp2p — сеть (Level 2)

### WireGuard
- Шифрованные P2P-туннели между устройствами
- Каждое устройство имеет постоянный внутренний IP в личной сети
- Пробивает NAT (через Tailscale-подобный механизм)

### libp2p
- Поиск узлов в децентрализованной сети
- Gossip протоколы для распространения данных
- DHT для адресации контента

### mDNS
- Обнаружение устройств в локальной сети
- Без интернета — прямая передача по Wi-Fi

## Резюме: почему не "всё на одном языке"

| Проблема | Решение |
|----------|---------|
| Нужна безопасность памяти | Rust |
| Нужна скорость JS-рантайма | Bun |
| Нужна нативная скорость в TS | TSCLANG |
| Нужна изоляция приложений | V8 Isolates |
| Нужна быстрая графика | WebGPU/Skia |
| Нужна распределенная сеть | WireGuard + libp2p |

Каждый слой делает одно дело и делает его лучше всех.
