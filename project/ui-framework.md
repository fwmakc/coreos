# CORE OS — UI-фреймворк

## Принцип: три уровня свободы

Разработчик выбирает уровень контроля — от полной свободы до строгих системных компонентов.

---

## Уровень 1: Raw Graphics (Полная свобода)

### Что дает
Прямой доступ к WebGPU/Skia контексту.

### Возможности
- Своя оболочка (альтернативный "рабочий стол")
- VR-интерфейсы
- Воссоздание Windows 95 внутри CORE
- 3D-редакторы, симуляции, тяжелые игры

### Для кого
Создатели альтернативных оболочек, тяжелые игры, специализированный визуальный софт.

---

## Уровень 2: Shell API (Свободные окна в стиле CORE)

### Что дает
Window Manager API: управление окнами, фокусом, слоями.

### Возможности
- Любая внутренняя логика и дизайн
- Окна подчиняются общим правилам: сворачивание, перетаскивание, примагничивание
- Свои брендбуки (Spotify, Discord, VS Code)

### Для кого
Крупные игроки с устоявшимся визуальным стилем.

---

## Уровень 3: Core Design System (Официальный UI)

### Что дает
Готовая библиотека компонентов: кнопки, списки, формы, блюр-панели.

### Киллер-фича: Автоматическая адаптация
Разработчик пишет `<Button>`, а система сама решает:
- "Офисный" режим — кнопка строгая
- "Киберпанк" тема — светится неоном
- На телефоне — больше под палец
- На десктопе — меньше под мышь

### Для кого
90% обычных приложений (заметки, калькуляторы, менеджеры задач). Выглядят идеально согласованно.

---

## Синтаксис (Developer Experience)

JSX остается, но вместо HTML-тегов — компоненты графического движка:

```typescript
import { View, Text, Button } from '@core/ui';

export default function MyApp() {
  return (
    <View style={{ layout: 'flex', gap: 10 }}>
      <Text value="Привет, это CORE!" style={{ color: '#fff' }} />
      <Button
        label="Нажми меня"
        onClick={() => os.notify('Клик!')}
      />
    </View>
  );
}
```

### Почему не HTML/CSS
- DOM = дерево из тысяч объектов, изменение пикселя → пересчет страницы (Layout Thrashing)
- CORE: "Нарисуй прямоугольник здесь, текст — там" — напрямую через WebGPU
- В 10-20x быстрее DOM
- Стриминг интерфейса: набор графических команд можно передать на другое устройство

---

## Слои рендеринга

```
1. Холст (WebGPU/Skia)           ← Уровень 1 рисует сюда
2. Окна (Window Manager)         ← Уровень 2 размещает здесь
3. Компоненты (Design System)    ← Уровень 3 отрисовывает внутри окон
```

Смена "Скина системы" — мгновенная трансформация от системного меню до последней кнопки.

---

## Нативный инспектор (Live Debugger)

Поскольку всё на JS: `Ctrl+I` в любом месте системы:
- Дерево компонентов (как в браузере)
- Изменение цвета/логики на лету
- Мгновенное применение

---

## Island Mode (для веб-контента)

Для обычных сайтов — изолированный модуль с Chromium-движком, работающий только внутри одного окна.

Система и нативные приложения летают на сверхлегком движке. Веб — в своей песочнице.

---

## Дизайн-киты и оболочки

### Системный магазин стилей
Один разработчик выпускает дизайн-пак — другие приложения подхватывают автоматически.

### Голосовое управление UI
"Сделай все окна в стиле терминала Fallout" → система переключает UI-слой.

### Концепция "Битвы оболочек"
Как KDE vs GNOME в Linux, но на CORE:
- Гики меряются UI-дизайнами
- Лучший бесплатный R&D для системы
- "Твой компьютер — твое отражение"
